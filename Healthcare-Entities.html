Healthcare Entities

A Healthcare Entity (HCE) refers to any licensed healthcare provider or organization that is authorized by its respective state to deliver professional healthcare services. This includes:

  • Individual providers licensed to practice healthcare
  • Healthcare organizations employing licensed professionals
  • eHealth Exchange organizations recognized by the state for providing licensed healthcare services

A healthcare entity can only submit patient data requests and receive responses through RxCheck after it has been officially onboarded into the state's RxCheck system.

HeadingDescription
NameThe name of the healthcare entity (HCE).
CodeA six-character code: two letters for the state code, an underscore (_), and three letters for the HCE site.
StatusIndicates the current status: Active, Inactive, or Firewall Pending.
Integration TypeEHR, EMR, HIE, or PDS (Pharmacy Dispensing System).
VendorThe integration vendor for the healthcare entity.
Site AddedDate and time when the HCE site was created.

9.1 Add a New Healthcare Entity Site

  1. Click on the Healthcare Entities button, located on the left-hand side of the screen.
  2. Click on the + Add Site button.
  3. Populate the Site Configuration Details for the new healthcare entity. Refer to the "Breakdown of Healthcare entity site details" section below for field-specific guidance.
  4. To populate the "Select Vendor" field, you must first click on the small lock symbol within the field.
  5. To exit without saving, click the Cancel button.
  6. To save the populated site information, press the Save button.
  7. Wait patiently as your site is being created.
  8. Populate additional details as requested in the corresponding data fields for each section.

9.2.3 Vendor Details

This section contains the SRS hosting details and indicates if the healthcare entity's integration is managed by the HCE IT team, the state, or by a vendor.

  • HCE IT — The IT team of the HCE oversees the SRS hosting and managing responsibilities.
  • State — The IT team of the state oversees the SRS hosting and managing responsibilities.
  • Vendor — A third-party vendor oversees the SRS hosting and managing responsibilities.
Note: If the Vendor option is chosen, additional fields need to be populated.
HeadingDescription
Where is it HostedHCE IT INFRA, Vendor IT INFRA, Private Cloud, or Government Cloud.
Vendor NameThe name of the vendor hosting the SRS.
Vendor AddressThe address of the vendor hosting the SRS.
Vendor ContactThe name of the contact responsible for managing the SRS.
Are the servers being accessed outside the US?A Yes or No question asking if the SRS will be accessed outside the United States.

9.2.4 Manage Roles

This section enables the State PDMP Administrator to assign roles that authorize specific users within a healthcare entity to submit prescription data requests. Roles are assigned by selecting the appropriate tags associated with the healthcare entity.

Clicking on a role changes it to green (active / "Selected"). An off-white color indicates the role is "Authorized" but inactive and cannot initiate a query.

ButtonFunctionality
Assign AllAutomatically selects all displayed roles.
Clear AllAutomatically deselects all displayed roles.
Show ListDisplays all available roles in two separate lists: Authorized Roles and Selected Roles.
Hide ListDisplays all available roles as tags (default view).

When in the list view, a blue arrow transfers a role from the Authorized Roles list to the Selected Roles list. An orange arrow transfers a role from Selected Roles back to Authorized Roles.

9.2.5 Manage Facilities

This section provides details of the facilities related to this specific healthcare entity.

9.2.5.1 How to Add a Single Healthcare Facility

  1. Click on the Add Facility button.
  2. In the pop-up window, enter the requested information in the appropriate data fields.
  3. Click on the Save button to save the populated information.
HeadingDescription
NameIndividual facility's name.
Facility TypeEHR or PDS.
Facility CodeThe HCE code is auto-populated; the facility code is entered as an alphanumeric value.
First NameContact person's first name at this individual facility.
Last NameContact person's last name.
EmailContact person's email.
PhoneContact person's phone number.
ExtensionContact person's phone extension, if any.

9.2.5.2 How to Add Multiple Healthcare Facilities

  1. Click on the Add Multiple Facilities button.
  2. You will be directed to the Add Facilities screen.
  3. Click on the Standardized Facilities Template link to download a template to populate (Excel .xlsx format). The first tab contains columns to enter facility information; the second tab titled Field Definitions provides an abbreviated summary of each field.
  4. Populate the template with each facility occupying a single row.
  5. Press the Click to select file button to choose the populated template to upload.
  6. If all facilities share a single contact person, check the box labeled One contact for all facilities. Note: If contact information is included in both the uploaded file and the UI, the UI entry will override and apply to all created facilities.
  7. Click the Upload button to import the data from your selected file.

9.3 Open a Healthcare Entity Record

  1. Click on the Healthcare Entities button, located on the left-hand side of the screen.
  2. Find the healthcare entity by:
    • Scrolling through the list on the screen, or
    • Typing the name of the HCE into the Global Filter search bar in the top right.
  3. Select the desired healthcare entity to view further details.
  4. To edit a record, make your changes and click the Save button. To exit without saving, click Cancel.

9.4 Export a List of Your HCEs and Facilities

  1. Click on the Healthcare Entities button, located on the left-hand side of the screen.
  2. In the top-right corner, click:
    • Export HCE's — Downloads a list of healthcare entities, their HCE code, and their status.
    • Export Facilities — Downloads a list of Facilities, their site code, and their status.
    Note: The HCE and facility lists are downloaded in a zipped folder.
  3. Right-click on the zipped folder to view options.
  4. Select the "Extract All…" option.
