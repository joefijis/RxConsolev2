Provider Validation

The term "Provider" refers to a licensed healthcare professional or organization legally authorized to deliver medical services to the public. Most providers in the US are uniquely identified by a 10-digit National Provider Identifier (NPI), issued by the Centers for Medicare and Medicaid Services (CMS). Additionally, the Drug Enforcement Agency (DEA) assigns practitioners a DEA Registration Number, authorizing them to prescribe controlled substances. Providers must also hold a valid state license, evidenced by a State License Number (SL#).

The Provider Validation feature in RxConsole allows State PDMP Administrators to configure whether the validation of the NPI, DEA, SL# and/or OTHER# is required, optional, or not validated prior to processing a request.

15.1 View and Add a Datasource for Provider Validation

  1. Click on the Provider Management button, followed by the Provider Datasource option, located on the left-hand side of the screen.
  2. The Datasources screen allows you to view previously uploaded datasources.
  3. To add a new provider datasource, press the Add Datasource button.
  4. To view a previously entered datasource, click on a datasource name in the List of Datasources.

15.2 View, Delete, and Search an Existing Datasource

Follow steps 1, 2, and 4 under the section "View and add a datasource for provider validation" to view an existing datasource. The new screen will look similar to the Add Datasource screen, but includes a Search Provider section.

To check if an existing datasource contains a provider, enter the DEA#, NPI#, SL# or OTHER# in the Search Provider section. If the provider is on the list, their information will be returned. If not, you will see "No Provider found."

Note: The datasource search feature is only available for datasource files and cannot be used on a datasource API.

15.2.1 Create a Datasource File

  1. Open Microsoft Excel on your computer.
  2. Create a new blank workbook.
  3. Add your desired headers in any order you prefer. The possible headers are: First Name, Last Name, NPI#, DEA#, SL#, OTHER#. Note: Not all header names are required — any combination can be used.
  4. Populate the relevant information under the corresponding column.
  5. When finished, save the file as either a .csv (Comma Separated Values) or .xlsx (Excel workbook).

15.2.2 Add a New Datasource File

  1. Follow steps 1–3 under "View and add a datasource for provider validation."
  2. Select the "File" option under the Datasource Type dropdown.
  3. Enter a name for your datasource file in the field labeled Datasource Name. Note: The name can be up to 15 alphanumeric characters (spaces and special characters are not allowed).
  4. Press the Select File button under the Data Upload section and find your datasource file.
  5. Select your import type: Full (imports the entire datasource file) or Delta (only available when viewing a previous datasource).
  6. Check the fields that are present in your datasource file. After checking a box, set the sequence (Column A = "0", Column B = "1", etc.).
  7. Press the Save button to add your datasource file to the RxConsole.
  8. Verify your datasource file was successfully added by looking for the alert: "Provider Data is Uploaded".

15.2.3 Add a Datasource API

  1. Follow steps 1–3 under "View and add a datasource for provider validation."
  2. Select the "API" option under the Datasource Type dropdown.
  3. Enter the API URL into the field labeled API URL.
  4. Select the appropriate authorization type using the Auth Type dropdown.
  5. For BASIC authorization, enter the username and password. For OAUTH2 authorization, enter the Client ID, Client Secret, and Auth URL.
  6. Press the Save button to add your datasource API.
  7. Verify the datasource API was successfully added by looking for the alert: "Datasource Created".

15.3 Configure Provider Validation Options

15.3.1 Modify an Existing Provider Validation

  1. Click on the Provider Management button, followed by the Provider Validation option.
  2. To view and/or modify an existing validation, click on the name of a validation.
  3. The screen allows you to change settings including:
    • Name — Change the name of the validation settings.
    • Select Roles — Change the roles that are subject to validation.
    • Validation Type — If Mandatory is chosen, a query will fail if the fields are not present. If Validate is chosen, the query stops if the validation check fails.
    • Datasource — Select a datasource to reference for the validation check (only present if Validate is chosen).
    • Selection — All checks all selected validation fields; Any requires only one to pass.
    • Validation Field(s) — Choose which ID numbers are subject to validation.
  4. Press the Save button to record your changes.
  5. Verify the update by looking for the alert: "Provider Validation Updated".

15.3.2 Add a New Provider Validation

  1. Click on the Provider Management button, followed by the Provider Validation option.
  2. Click on the Add Provider Validation button.
  3. Enter the settings for the new validation rules (Name, Select Roles, Validation Type, Datasource, Selection, and Validation Fields).
  4. Press the Save button to add your new provider validation rules.
  5. Verify the creation by looking for the alert: "Provider Validation Created".
